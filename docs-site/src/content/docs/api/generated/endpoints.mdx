## Health

### GET /health

Get service health status

Returns overall service health including account connections, dead letter queue size, and optional integration statuses (Tika, ClamAV). No authentication required.

**Source:** `src/server/health.ts`

**Response Example:**
```json
{
  "status": "ok",
  "uptime": 3600,
  "accounts": {
    "connected": 2,
    "total": 2
  },
  "deadLetterCount": 0,
  "lastProcessed": "2026-01-21T10:30:00.000Z"
}
```
